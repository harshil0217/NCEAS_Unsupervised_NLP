# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Environment

Requires Linux + CUDA GPU (`phate-env` conda environment). cuML, cuPCA, cuUMAP, cuHDBSCAN are GPU-only and will fail on macOS/Windows. Activate with:

```bash
conda activate phate-env
```

## Running the Pipeline

All scripts are run from the **repo root**, not from inside `src/`:

```bash
# Benchmark datasets (rcv1 is smallest — recommended for testing)
python src/run_models/benchmark_datasets/eval_pipeline.py --dataset rcv1
python src/run_models/benchmark_datasets/eval_pipeline.py --dataset arxiv
python src/run_models/benchmark_datasets/eval_pipeline.py --dataset amazon
python src/run_models/benchmark_datasets/eval_pipeline.py --dataset dbpedia
python src/run_models/benchmark_datasets/eval_pipeline.py --dataset wos

# Hercules clustering (benchmark datasets)
python src/run_models/benchmark_datasets/herc_pipeline.py

# Synthetic data: generate + embed + cluster (all in one)
cd src/run_models/synthetic_data && bash run_all.sh

# Scatter grid figures (synthetic, 2D visualization)
python src/run_models/synthetic_data/scatter_grid_synthetic.py

# Visualization metrics + Shepard diagrams
python src/run_models/synthetic_data/viz_metrics_script.py
python src/run_models/benchmark_datasets/viz_metrics_script.py
```

## Critical Path Convention

Every script under `src/` uses this pattern at startup:

```python
# Walk up to src/, then chdir there
os.chdir(src_dir)
sys.path.insert(0, src_dir)
```

This means **all paths inside scripts are relative to `src/`**:
- Data inputs: `../data/<dataset>/`
- Cached intermediates: `../data/intermediate_data/`
- Final outputs: `../results/`

When adding new file I/O to any script under `src/`, always prefix with `../`.

## Architecture

### Pipeline stages (in order)

1. **Embed** — texts → `.npy` embeddings via `SentenceTransformer`
   - Two models: `sentence-transformers/all-MiniLM-L6-v2` and `Qwen/Qwen3-Embedding-0.6B`
   - Cached in `data/intermediate_data/{model}_embeddings/`

2. **Dimensionality reduction** — embeddings → reduced `.npy`
   - **300-dim** for clustering: PHATE, PCA (cuPCA), UMAP (cuUMAP), tSNE (cuTSNE), PaCMAP, TriMAP
   - **2-dim** for visualization: same methods, separate cache in `{model}_reduced_2d/`
   - Cached in `data/intermediate_data/{model}_clustering_results/` (or `_reduced_2d/` for 2D)

3. **Cluster** — at each hierarchy level
   - Agglomerative (ward linkage via scipy, then `fcluster`), HDBSCAN, Diffusion Condensation, Hercules
   - Linkage matrices cached in `data/intermediate_data/{model}_linkage/`
   - Per-level scores cached as `.npy` in `data/intermediate_data/{model}_scores/`

4. **Evaluate** — clustering labels vs. ground truth
   - Metrics: FM index, Rand, ARI, AMI, Dendrogram Purity, LCA-F1
   - Ground truth trees (anytree) pre-built as `.pkl` in `data/intermediate_data/ground_truth_trees/`
   - Final CSVs → `results/{dataset}_clustering_scores.csv`

### Caching behavior

All expensive steps check for a cached `.npy` or `.pkl` before recomputing. Scripts are safe to rerun and will resume from where they left off.

### Custom packages (`src/custom_packages/`)

These are local implementations imported via the `sys.path` trick above:
- `hercules.py` — LLM-based hierarchical k-means (Groq API)
- `diffusion_condensation.py` — DC clustering
- `fowlkes_mallows.py`, `dendrogram_purity.py`, `lca_f1.py` — custom evaluation metrics
- `graph_utils.py` — utilities for converting scipy linkage trees to anytree

### Output locations

| Output type | Location |
|-------------|----------|
| Clustering scores (CSV) | `results/` |
| Scatter grid PNGs | `results/summary_figures/` |
| Shepard diagram PNGs | `results/shepard_diagrams/{model}/` |
| Embeddings (cache) | `data/intermediate_data/{model}_embeddings/` |
| 2D reductions (cache) | `data/intermediate_data/{model}_reduced_2d/` |
| Clustering reductions (cache) | `data/intermediate_data/{model}_clustering_results/` |
| Per-level metric scores (cache) | `data/intermediate_data/{model}_scores/` |
| Linkage matrices (cache) | `data/intermediate_data/{model}_linkage/` |
| Ground truth trees (cache) | `data/intermediate_data/ground_truth_trees/` |
| Viz metrics CSVs (cache) | `data/intermediate_data/{model}_results/` |

`data/intermediate_data/` is entirely `.gitignore`d except for its `README.md`.
